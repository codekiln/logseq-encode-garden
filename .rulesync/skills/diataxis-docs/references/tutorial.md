# Diataxis Tutorials in Logseq

Tutorials are **learning-oriented** guides that provide a hands-on experience. A tutorial takes the learner through meaningful practical steps, focuses on learning by doing, builds confidence through successful completion, and minimizes explanation in favor of action.

## Key Principles
1. **Focus on Learning** — help users acquire skills through guided practice
2. **Show Early Results** — deliver visible achievements frequently
3. **Minimize Explanation** — keep focus on doing; link to explanations elsewhere
4. **Ensure Success** — every step must work reliably for every user
5. **Be Concrete** — specific examples, avoid abstractions
6. **Guide Completely** — take full responsibility for the learner's success

## Structure (LFM)

```
tags:: [[ExistingTag1]], [[ExistingTag2]], [[Diataxis/Tutorial]]

- # Tutorial: [Learning Goal]
  - ## What You'll Create
    - Brief description of the end result
  - ## Prerequisites
    - Required tools or knowledge; setup steps if needed
  - ## Learning Goals
    - What skills they'll acquire; what they'll be able to do afterward
  - ## Steps
    - ### 1. First Achievement
      - Clear, concrete instructions; expected results; what to notice
    - ### 2. Second Achievement
      - And so on...
  - ## What You've Learned
    - Recap of skills acquired; next steps for practice
  - ## Related
    - Links to related pages
```

## Naming and File Paths
Namespace pattern: `Topic/Subtopic/Tutorial/Learning Goal`.
- `[[Programming/Python/Tutorial/Build Your First Web App]]` → `pages/Programming___Python___Tutorial___Build Your First Web App.md`

## Required Tags
All Diataxis Tutorial pages MUST include `[[Diataxis/Tutorial]]` in frontmatter. Add it to existing tags; never remove existing tags.

## Language Guidelines
- Use "we": "We will create..."
- Present tense, active voice
- Clear expectations: "You will see..."
- Point out observations: "Notice that..."
- Keep explanations minimal; link to reference material
- Focus on concrete actions and results

## Writing Process
1. Plan the learning journey — identify skills, break into small achievable steps, each producing visible results.
2. Write — clear learning goals, concrete actions, expected results, what to notice.
3. Test — follow your own instructions exactly; verify each step works reliably.
4. Review — remove unnecessary explanations, add missing steps, ensure consistent success.

A good tutorial takes full responsibility for the learner's success, provides a reliable repeatable experience, builds confidence through early/frequent achievements, and favors concrete actions over abstractions.
