# Rulesync: canonical sources only (agents)

## Non-negotiable

- **Canonical edits:** Change **only** files under **`.rulesync/`** — `rules/`, `commands/`, `skills/`, `subagents/`, `config/`, and **`rulesync.jsonc`** when targets or features change.
- **Forbidden:** Do **not** hand-edit anything **`rulesync generate`** writes for Cursor, Claude Code, Copilot, Codex CLI, or other configured targets. Treat those paths as **read-only compiled output**.
- **When something looks wrong in a tool copy:** Fix the matching file under **`.rulesync/`**, then run **`rulesync generate`** (or **`npx rulesync generate`**) from the **repository root**. Do **not** patch the generated file in place.

## Examples of generated output (do not edit directly)

Non-exhaustive — this repo also documents many of these under **“Generated by rulesync”** in **`.gitignore`**:

- **Cursor:** `.cursor/rules/*.mdc`, `.cursor/skills/**` (and related Cursor outputs rulesync emits)
- **Claude Code:** `.claude/rules/**`, generated `.claude/skills/**`, commands/agents/memories paths your `rulesync.jsonc` + version populate
- **GitHub Copilot:** `.github/instructions/**`, `.github/copilot-instructions.md`, generated `.github/skills/**`, `.github/agents/**`, etc., per configuration
- **Codex / other targets:** e.g. `.codex/**` when generated
- **Root:** **`AGENTS.md`** when rulesync generates it from this project’s sources

If a path is produced by **`rulesync generate`**, it is **not** the place to fix wording or behavior—change the **`.rulesync/`** source and regenerate.

## After you change `.rulesync/`

1. Run **`rulesync generate`** from the repo root (same flags as usual for this project; default is fine if you use **`rulesync.jsonc`**).
2. Commit **both** the **`.rulesync/`** change and any **tracked** generated files this repo expects in git (per **`rulesync-overview`** and **`.gitignore`**).
