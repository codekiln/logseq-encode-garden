- # Every Tool Needs an Alibi
	- Compared what's actually brew-installed on this machine against `chezmoi/Brewfile` — a long list of formulae and casks (logseq, 1password-cli, pyenv, tfenv, fig, postgresql@14, ...) installed but never declared. Wrote a companion pair of Ideas for [[My/Dotfiles/Process/Adding a New Global Dependency/Idea]]: an entry gate that already exists (nothing installed without a written why) needs an exit gate too (nothing stays installed without a *recalled* why).
	- Added a Try-before-commit window subsection to that Idea for the Homebrew case specifically — installing with `brew install x` and declaring `x` in the Brewfile are separate actions, which is exactly the low-friction trial the Idea already wants; the ritual is to compare against the Brewfile weekly and uninstall what isn't declared, per [[My/Principle/Declarative over Imperative]].
	- Then a more speculative companion idea: use spaced repetition — the same mechanism already behind [[Logseq/Flashcard/Review/Favorite]] decks for [[mise]]/[[chezmoi]]/[[tmux]]/[[yazi]] — as the retention gate itself. Fail to explain why something's installed enough times, it gets pruned. Filed as its own Idea rather than folded in, since it works by memory checks instead of an intake conversation.
