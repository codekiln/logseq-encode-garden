- # Every Tool Needs an Alibi
	- Went through the drift between `brew list` and `chezmoi/Brewfile` on this machine — a pile of formulae and casks (logseq, 1password-cli, pyenv, tfenv, fig, postgresql@14, ...) installed on request but never declared. Landed on a companion pair of Ideas for [[My/Dotfiles/Process/Adding a New Global Dependency/Idea]]: an entry gate that already exists (nothing installed without a written why) needs an exit gate too (nothing stays installed without a *recalled* why).
	- Added a Try-before-commit window subsection to that Idea for the Homebrew case specifically — `brew install x` is decoupled from declaring `x` in the Brewfile, which is exactly the low-friction trial the Idea already wants; the ritual is a weekly diff-and-uninstall against the Brewfile, per [[My/Principle/Declarative over Imperative]].
	- Then a wilder companion idea: use spaced repetition — the same mechanism already behind [[Logseq/Flashcard/Review/Favorite]] decks for [[mise]]/[[chezmoi]]/[[tmux]]/[[yazi]] — as the retention gate itself. Fail to explain why something's installed enough times, it gets pruned. Filed as its own Idea rather than folded in, since it's a distinct mechanism (recall-gated retention) from the intake process.
