# continued studies
	- ## in [[mise]]
		- reviewed [[mise/use]] vs [[mise/install]] again. `mise use` does a `mise install` and then also adds it in `mise.toml`
		- still a bit confused on [[mise/plugin]]s vs [[mise/Backend]]s. I get the sense that both are systems for using an external tool, but mise backends are written in rust and therefore have more of a "native mise" support system, while mise plugins are an older concept designed to shell out to asdf plugins or vfox plugins.
	- ## nix weirdness
		- filled in [[Nix/Package/Registry]] with a reference to search.nixos.org, which felt wrong. this is an example of why nix has bad [[DX]]: it chooses uses the search interface as the package registry, instead of saying that nix has a package registry that you can search. It's a small conceptual leap, but conflating the package registry with the package search interface just confuses people.
