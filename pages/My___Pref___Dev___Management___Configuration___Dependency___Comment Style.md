# Dependency Comment Style
	- Deviations from [[My/Pref/Dev/Management/Configuration/Comment Style]] for dependency *declarations* — the entries an install manifest lists.
	- Beyond the *why*, a dependency entry also names *what* the dependency is.
	- When a dependency isn't handled by the preferred manager for its kind, the comment also says *why it lives where it does* (e.g. "GUI cask, not installable via mise").
