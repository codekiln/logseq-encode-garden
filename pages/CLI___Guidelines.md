- # [Command Line Interface Guidelines](https://clig.dev/)
	- An open-source guide to writing better command-line programs, adapting traditional Unix conventions for modern CLI design.
	- ## Scope
		- Covers help text, documentation, output, errors, arguments and flags, interactivity, subcommands, robustness, future-proofing, signals, configuration, environment variables, naming, distribution, and analytics.
	- ## Maintainer
		- Written by Aanand Prasad, Ben Firshman, Carl Tashian, and Eva Parish; design by Mark Hurrell.
		- Source: [cli-guidelines/cli-guidelines](https://github.com/cli-guidelines/cli-guidelines)
