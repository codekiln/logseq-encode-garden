- # [PTKM](https://ptkm.net/home)
	- ## Overview
		- **PTKM** (Project, Task, and Knowledge Management) is an integrated framework for managing projects, tasks, and knowledge across work and life—positioned as a full system rather than a single app feature set.
		- Public documentation and articles live at [ptkm.net](https://ptkm.net/home); the site is published with [Obsidian Publish](https://publish.obsidian.md).
	- ## Context
		- Sits alongside [[PKM]] and “second brain” practice: emphasizes **tasks as the unit of work** (capture, meetings, reading, writing, scheduling, delivery) while keeping **notes and links** as durable context.
		- Authored and maintained by [[Person/wenlzhang]], who also ships [[Obsidian]] community plugins that extend the PTKM workflow.
	- ## Key Principles
		- **Task-centered workflow** — Treat capturing thoughts, meetings, reading, writing, knowledge-base work, scheduling, and delivery as tasks in one flow.
		- **Providing context** — Write tasks and notes for a future self or stranger so nothing is orphaned later.
		- **Linking everything** — Relate tasks, projects, notes, people, and other entities so context stays navigable.
		- **Focusing on important matters** — Explicit prioritization because not everything can be done.
		- **Redundancy for reliability** — Build trust in processes through deliberate backup and safety (e.g. link maintenance when reorganizing notes).
	- ## Mechanism
		- **Learn:** overview on the home site; deeper tutorials under the [Service](https://ptkm.net/) area.
		- **Implement:** choose tools that support linking, tasks, and long-horizon projects; the PTKM site treats **[[Obsidian]]** as the primary implementation environment (rich core features, plugin ecosystem, canvas, Mermaid Gantt for long projects).
	- ## [[Obsidian]] implementation
		- PTKM documentation recommends **[[Obsidian]]** vaults for day-to-day PTKM practice and hosts its own docs via Obsidian Publish.
		- Companion **Obsidian plugins** (documented on ptkm.net) extend task status, Todoist bridging, canvas sends, tag indexes, timestamp links, link maintenance, URI conversion, folder navigation, and title sync—see the site’s “How to Implement PTKM” section for the current list.
		- In this garden: [[obsidian-file-title-updater]] (**File Title Updater**) by [[Person/wenlzhang]] syncs filename, frontmatter title, and first `#` heading; docs at [ptkm.net/obsidian-file-title-updater](https://ptkm.net/obsidian-file-title-updater).
	- ## Examples
		- Featured themes on the site include sustainable deep-work productivity, second-brain maintenance cost, bridging tasks and knowledge (e.g. Todoist Context Bridge), complex evolving tasks, and long-term project Gantt charts in Obsidian.
	- ## Misconceptions
		- “PTKM is only an Obsidian plugin collection” — **False**; it is a **methodology** with optional Obsidian tooling, not a single product binary.
		- “PTKM replaces [[PKM]]” — **Oversimplified**; PTKM **narrows** PKM toward **projects and tasks** with explicit principles and implementation guidance.
