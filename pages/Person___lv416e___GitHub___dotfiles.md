- # [lv416e/dotfiles: Modern dotfiles management with chezmoi](https://github.com/lv416e/dotfiles)
	- Public **[[dotfiles]]** for [[Person/lv416e]], managed with **[[chezmoi]]** (templated files, optional **age** and **1Password CLI** for secrets). After `chezmoi apply`, setup continues with **`brew bundle`** from the chezmoi-managed Brewfile and **`mise install`** / **`mise trust`** (see upstream README).
	- **Homebrew** — `dot_Brewfile.tmpl` drives a large CLI stack (**[[mise]]**, **[[chezmoi]]**, **[[jj-vcs]]**, **[[nvim]]**, **[[Docker]]**, **[[ripgrep]]**/[[fzf]]/[[fd]]/[[eza]]/[[bat]], **[[starship]]**, **[[tmux]]**, **[[zellij]]**, **[[ghq]]**, etc.) and, on **macOS** (`darwin`), conditional **casks** (terminals, editors, daily apps). Useful as a reference layout for “modern Rust/CLI + GUI” Brew bundles.
	- **[[tmux]] / [[zellij]]** — [docs/reference/keybindings-reference.md](https://github.com/lv416e/dotfiles/blob/main/docs/reference/keybindings-reference.md) documents **[[Keyshort]]**-style bindings and an **equivalency table** between **[[zellij]]** and **[[tmux]]** (same mental model across their multiplexer abstraction).
	- **[[mise]]** — universal tool versions and tasks (`mise sync`, `mise a`, docs for tasks in-repo).
	- **[[fnox]]** — [docs/guides/secrets-management-fnox.md](https://github.com/lv416e/dotfiles/blob/main/docs/guides/secrets-management-fnox.md) documents **[[fnox]]** from [[Person/Jeff Dickey]] (jdx), the same ecosystem as **[[mise]]**, for secrets in this dotfiles layout (complements **age** / **1Password CLI** paths above).
	- **[[DevContainer]]** — repo includes `.devcontainer/` (`devcontainer.json`, `Dockerfile`, template metadata, scripts) for containerized environments aligned with the same toolchain story.
		- Their chezmoi **`run_onchange`** hook [run_onchange_install-packages.sh.tmpl](https://github.com/lv416e/dotfiles/blob/main/run_onchange_install-packages.sh.tmpl) documents **`brew bundle`** from `~/.Brewfile` for **macOS, Linux, Codespaces, and Dev Container** (after a prior `run_once_before` installs Homebrew — on Linux/Dev Container style hosts it wires **`linuxbrew`** from `/home/linuxbrew/.linuxbrew/bin/brew`). So **Homebrew is part of their intended path even inside dev containers**, not only on the Mac.
		- [docs/guides/codespaces-prebuilds-setup.md](https://github.com/lv416e/dotfiles/blob/main/docs/guides/codespaces-prebuilds-setup.md) — worthwhile upstream notes on **[[GitHub/Codespace]] prebuilds** wired for their **[[DevContainer]]** story, including how **Homebrew** fits that Codespaces / container setup (not only interactive installs).
		- [docs/guides/devcontainer-setup-complete.md](https://github.com/lv416e/dotfiles/blob/main/docs/guides/devcontainer-setup-complete.md) — practical guide for keeping **[[DevContainer]]** and **[[GitHub/Codespace]]** support in sync (same mental model for local containers and cloud Codespaces).
	- **Architecture Decision Records** — [docs/adr](https://github.com/lv416e/dotfiles/tree/main/docs/adr) documents dotfiles design using **[[Architecture/Decision/Record/MADR]]** (their [README](https://github.com/lv416e/dotfiles/blob/main/docs/adr/README.md) links the official [MADR template](https://adr.github.io/madr/) and lists nine **Accepted** ADRs `0001`–`0009`: [[jj-vcs]] alongside Git, zsh-defer, dual encryption, multiplexer abstraction, configuration variants, Rust ecosystem adoption, Difftastic, Linux best-effort support, macOS defaults automation (dates **2025-10-26**–**2025-11-01** per index).
