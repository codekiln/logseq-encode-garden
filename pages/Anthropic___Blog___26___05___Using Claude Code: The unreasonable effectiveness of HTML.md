- # [Using Claude Code: The unreasonable effectiveness of HTML](https://claude.com/blog/using-claude-code-the-unreasonable-effectiveness-of-html)
	- ## Summary
		- Argues that HTML is often a better output format than Markdown for Claude Code artifacts because it supports richer layout, clearer visual structure, easier sharing, and lightweight interactivity.
		- Frames HTML artifacts as a way to stay more "in the loop" with Claude when reviewing plans, specs, explainers, and other long-form agent output.
	- ## Notes
		- Reasons given: higher information density, better readability for long docs, easier browser-based sharing, two-way interactions, and stronger data ingestion from local files, MCPs, browser context, and git history.
		- Suggested use cases: specs and planning, code review and understanding, design prototypes, reports and explainers, and throwaway task-specific editors that export results back into prompts or repo files.
		- Recommended starting point is simple: ask Claude Code to "make an HTML file" or "make an HTML artifact", then turn recurring patterns into reusable skills later.
		- Byline: Thariq Shihipar.
