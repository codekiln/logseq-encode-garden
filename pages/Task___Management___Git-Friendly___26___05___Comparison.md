# Git-friendly task manager comparison
	- ## Evaluation criteria
		- A good replacement for this repo should store task state in Git-friendly text files or Git-native objects, avoid [[SQLite]] as the committed source of truth, and work from a CLI without requiring a hosted service.
		- It should be legible to AI coding agents without dumping huge JSONL blobs into context, support basic issue states and dependencies, and have a clear ready/blocked view for resuming work.
	- ## Candidates
		- In order of current preference, high to low
		- {{embed [[BeansTasks]]}}
		- {{embed [[beadwork]]}}
		- {{embed [[ff]]}}
		- {{embed [[tk]]}}
	- ## Excluded
		- [[OpenSpec]] - it's not really a task manager. It's only oriented at "what's the next change"
		- [[td]] and [[td/sidecar]] remain useful references, but this garden's definition of [[Task/Management/Git-Friendly]] rejects SQLite-backed task state as not truly Git-friendly for this purpose.
		- [[HamsterBase Tasks]] and [[Will Be Done]] are interesting local-first task apps, but they are GUI or app-sync oriented rather than CLI repo-local Beads replacements.
